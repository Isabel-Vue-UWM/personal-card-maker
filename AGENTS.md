## Behavior Guidelines for Copilot
When generating code or suggestions: 

- Prefer clarity over cleverness
- Comment complex logic briefly
- Avoid premature optimization
- Keep functions small and readable
- Use descriptive variable names
- Always assume a beginner might read this code later

When unsure, explain trade‑offs in comments rather than asking questions.

---

## README Expectations
Also help generate a `README.md` that covers: 
- What the app is
- How to use it
- How data is stored (local only)
- How to deploy on GitHub Pages

---

## Non‑Goals
Do **not**: 
- Add user accounts or authentication
- Send data anywhere external
- Track users or add analytics
- Introduce ads or monetization
- Over‑engineer the solution

---

## Tone
This is a **positive mental‑health–adjacent tool** (not medical).
Keep everything: 
- Gentle
- Encouraging
- Non‑judgmental

Copy should feel supportive and warm.

---

## Final Reminder
Your goal is to help the user **remember the good things in life** through gentle interaction, simple visuals, and dependable functionality.
